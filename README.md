# 📱 SMS Pro — 智能短信远程管理系统

> **版本：** Pro v1.0  
> **架构：** ESP32 + PHP 后端 + Web 管理面板  
> **核心能力：** 短信收发 · 远程控制 · 多通道推送 · 长短信合并 · 实时状态监控

---

## 🎯 产品定位

**SMS Pro** 是一款面向企业运维、IoT 设备管理、自动化通知场景的**专业级短信网关解决方案**。通过 ESP32 硬件模组连接 4G/2G 网络，实现短信的**接收转发**与**远程发送**，并配备完整的 Web 管理后台，支持多设备监控、队列管理、多渠道推送。

## ✨ 核心功能特性

### 1. 双模通信能力

| 模式 | 说明 |
|------|------|
| **接收模式** | 自动监听 `+CMT` 通知，实时解析 PDU 短信，支持长短信合并 |
| **发送模式** | 支持本地 Web 发送 + 远程服务器队列下发，带 ACK 确认机制 |

### 2. 长短信智能合并

- **自动分段识别**：基于 UDH 头部解析 `refNumber` / `partNumber` / `totalParts`
- **超时兜底机制**：30 秒未收齐自动合并转发，缺失部分标记为 `[缺失N]`
- **多会话并发**：支持 5 条独立长短信同时合并，每条约 10 条分片

### 3. 多渠道推送通知

| 通道 | 状态 | 配置方式 |
|------|------|---------|
| **QQ 邮箱 (SMTP)** | ✅ 支持 SSL | 面板配置服务器/端口/授权码 |
| **企业微信 Webhook** | ✅ 支持 | 面板配置机器人地址 |
| **远程服务器上报** | ✅ 支持 | 自动上报接收短信 + 心跳 |

### 4. 远程服务器控制（Pro 核心）

- **心跳上报**：每 60 秒上报设备状态（IP、信号强度、内存、时间戳）
- **轮询拉取**：每 15 秒拉取服务器待发送队列
- **ACK 确认**：设备收到任务后立即确认，防止重复下发
- **结果回执**：发送成功/失败实时上报服务器，带重试机制（3 次）
- **防重发机制**：50 条 ID 缓存环，避免重复处理同一短信

### 5. 专业 Web 管理面板

- **安全登录**：Session 鉴权，密码保护
- **实时状态**：设备在线/离线状态（120 秒心跳超时判定）
- **设备信息**：ID、IP、信号强度、剩余内存、最后心跳时间
- **发送管理**：待发送 / 发送中 / 已发送 / 发送失败 四态追踪
- **接收记录**：完整接收历史，支持一键清空
- **自动刷新**：1 秒间隔局部刷新，无需手动刷新页面

### 6. 本地设备配置界面

- **WiFi 扫描**：一键扫描附近网络，点击自动填入
- **配置持久化**：Preferences 存储，断电不丢失
- **智能重启**：WiFi 变更后自动重启设备
- **推送开关**：邮件 / 企业微信 / 远程控制 独立开关

## 📡 通信协议规范

### 设备 → 服务器（POST）

| 类型 | 字段 | 说明 |
|------|------|------|
| `heartbeat` | `device_id`, `device_ip`, `rssi`, `free_heap`, `unixtime` | 心跳包 |
| `received` | `sender`, `content`, `timestamp`, `device_ip` | 接收短信上报 |
| `ack` | `id`, `device_id` | 任务确认 |
| `result` | `id`, `success`, `message` | 发送结果回执 |

### 服务器 → 设备（GET Response）

```json
{
  "send": [
    { "id": "sms_xxx", "phone": "13800138000", "content": "测试内容" }
  ],
  "cmd": null
}
```

### 数据文件结构（`sms_data.json`）

```json
{
  "pending":  [ { "id", "phone", "content", "time", "status" } ],
  "sending":  [ { "id", "phone", "content", "time", "status", "ack_time" } ],
  "sent":     [ { "id", "phone", "content", "time", "status", "result_time", "result_msg" } ],
  "received": [ { "sender", "content", "timestamp", "device_ip", "report_time" } ]
}
```

## 🔐 安全设计

- **API 密钥鉴权**：所有设备通信携带 `key` 参数，服务器端校验
- **管理面板密码**：独立 Session 登录，支持登出
- **XSS 防护**：前端 `escapeHtml` 转义，后端 `htmlspecialchars` 输出
- **HTTPS 通信**：ESP32 使用 `WiFiClientSecure` + `setInsecure()` 支持 SSL
- **防重放攻击**：设备端 ID 缓存 + 服务器端状态机管理

## 🛠️ 技术栈

| 层级 | 技术 |
|------|------|
| **硬件** | ESP32-S3 / ESP32-C3 + 4G Cat.1 模组（UART） |

| **后端** | PHP 8.x, JSON 文件存储（无数据库依赖） |
| **前端** | 原生 HTML5 + CSS3 + JavaScript（无框架依赖） |
| **通信** | HTTP/HTTPS, RESTful API, JSON |

## ⚡ 快速开始

1. **硬件准备**：ESP32-C3开发板SuperMini ￥9.9 + 4G 模组 ML307R-DC AT固件 ￥16
2. **配置信息**：配置 `sms.ino` ①20-21排 `WiFi名称` / `WiFi密码`  ②32-33排 `你的邮箱号码` / `企业微信Webhook地址`  ③36排 远程服务器 域名
3. **烧录固件**：上传 `sms.ino`  网页统一密码：admin （可自行修改 ）
4. **部署后端**：将 `index.php` + `api.php` 上传至 PHP 服务器，确保写入权限
5. **配置远程**：在设备 Web 界面开启「远程控制」，填入服务器 URL 和 API Key
6. **开始使用**：访问 Web 面板发送短信，或接收短信自动推送
